# flight-meta

Метапоиск авиабилетов с **виртуальным интерлайном** (само-стыковка из отдельных
билетов) и **визово-транзитной логикой** под паспорта РФ/СНГ.

## Зачем

Агрегаторы (Aviasales и др.) показывают только маршруты, продающиеся как **единый
билет** в GDS. Они не «склеивают» два отдельных билета разных авиакомпаний —
поэтому реально существующие и часто более дешёвые маршруты (через любой хаб:
Стамбул, Дубай, Китай и т.п.) не видны. Этот сервис их находит.

Модель — **метапоиск без наценки пользователю**: находим, показываем, отправляем
покупать к авиакомпании/агентству (редирект). Доход — партнёрские отчисления.

Полный план и дорожная карта: `~/.claude/plans/greedy-snuggling-hummingbird.md`.

## Источник данных: свой комбайнер

Kiwi/Tequila недоступен, поэтому уникальные маршруты строит **собственный
комбайнер** (`internal/combiner`): берёт per-leg цены из `sources.LegSource`,
перебирает хабы и склеивает `A→хаб` + `хаб→B` в само-стыковочные офферы с
проверкой времени стыковки и транзит-визы. Реальный фид цен (например
Travelpayouts) реализует `LegSource` и встаёт на место `mockleg` без изменений
остального конвейера.

## Статус: Фазы 1–5 готовы (на mock-источнике плеч)

- **Фаза 1** — Go-сервис: модель оффера, интерфейс источника, fan-out, фильтры,
  ранжирование.
- **Фаза 2** — React-фронт (поиск → выдача, фильтры, бейджи, предупреждения).
- **Фаза 3** — транзит-визы РФ/СНГ (`internal/visa`): бейджи «без визы / TWOV /
  нужна виза», фильтр «только безвиз-транзит», airside vs само-стыковка, дисклеймер.
- **Фаза 4** — гибкие даты: `GET /calendar` — цена по дням ±N с пометкой
  самого дешёвого дня.
- **Фаза 5** — безопасное время стыковки (`internal/connection`): MCT-оценка
  safe/risky/infeasible (само-стыковка требует больше; виза-требующий landside —
  infeasible), фильтр «скрыть невозможные стыковки».

## Фронтенд (web/)

React + Vite + TypeScript. Карточки следуют паттернам flight-search (крупная
цена, таймлайн, пилюли пересадок, бейджи), на стыковках — бейджи транзит-визы и
риска стыковки; календарь подсвечивает «рядом дешевле».

**Тема и настройки** (`src/settings.ts`, шестерёнка в шапке): дефолт — «тёплая
журнальная» (крем + жжёный оранж + слива, серифные заголовки), плюс «тёмная
неоновая» и «необрутализм»; выбор шрифта, размера и плотности. Сохраняется в
localStorage. Веб-шрифты не грузятся (только системные) — быстро и без отправки
IP на иностранный CDN.

**Правовое (РФ)** — страницы `#/legal/*` (политика ПД, согласие, соглашение),
баннер согласия, реквизиты владельца в футере, маркировка партнёрских ссылок.
Полный чек-лист и операционные задачи — в [`COMPLIANCE.md`](COMPLIANCE.md).

```bash
# 1) бэкенд
go run ./cmd/api
# 2) фронт (Vite dev проксирует /api → :8080)
cd web && npm install && npm run dev   # http://localhost:5173
```

Прод-сборка: `cd web && npm run build`. База API настраивается через `VITE_API_BASE`
(по умолчанию `/api`, в dev — прокси на бэкенд).

## API

`GET /health` → `{"status":"ok"}`

`GET /search` — параметры:

| Параметр | Обяз. | Пример | Описание |
|---|---|---|---|
| `origin` | да | `MOW` | IATA-код вылета (3 буквы) |
| `destination` | да | `BKK` | IATA-код назначения |
| `depart` | да | `2026-04-20` | дата вылета `YYYY-MM-DD` |
| `return` | нет | `2026-04-27` | дата обратно (one-way, если пусто) |
| `passengers` | нет | `2` | 1..9, по умолчанию 1 |
| `passport` | нет | `RU` | страна паспорта — определяет транзит-визы |
| `currency` | нет | `RUB` | валюта цен |
| `stops` | нет | `direct` / `one` / `one_plus` | фильтр пересадок |
| `airlines` | нет | `TK,EK` | только эти авиакомпании |
| `exclude_airlines` | нет | `SU` | исключить авиакомпании |
| `self_transfer` | нет | `false` | выключить само-стыковочные варианты |
| `visa_free_transit` | нет | `true` | только варианты без транзитной визы |
| `hide_infeasible` | нет | `true` | скрыть невозможные стыковки (мало времени/виза) |

`GET /calendar` — те же параметры + `window` (1..14, дней в обе стороны). Ответ:
`{ days: [{ date, priceMinor, currency, hasOffers, cheapest }] }` — для подсказки
«рядом дешевле».

Ответ `/search`: `{ offers: [...], sources: [...], visaDisclaimer }`. У само-стыковочных
офферов `connection: "self_transfer"` и часто `unique: true`; у стыковок —
`visaStatus` (`no_visa`/`twov`/`visa_required`/`unknown`) и `transitNote`.

## Структура

```
cmd/api            точка входа сервиса
internal/offer     единая модель Offer (без зависимостей)
internal/sources   Adapter + Query/Filters; LegSource; mockleg/ — заглушка цен плеч
internal/combiner  свой комбайнер: склейка A→хаб→B в само-стыковки
internal/search    fan-out, мердж, обогащение (визы, стыковки), ранжирование, /calendar
internal/rank      фильтры (пересадки, а/к, само-стыковка, визы, стыковки) + сортировка
internal/visa      транзит-визы: embed-база правил РФ/СНГ, резолвер, бейджи
internal/connection MCT-оценка стыковок (safe/risky/infeasible)
internal/config    настройки из env (секреты только server-side)
internal/httpapi   HTTP/JSON, валидация запроса, security-заголовки
web/               React + Vite + TypeScript
deploy/            Dockerfile (далее — Helm для K8s)
```

> Транзит-визовые данные в `internal/visa/data/*.json` — **индикативные** (на 2026-06)
> и встроены в бинарь через `go:embed`. Это не официальная справка; позже сюда
> подключается авторитетный источник (Timatic/sherpa).

## Дорожная карта (кратко)

- Реальный фид цен плеч через `sources.LegSource` (Travelpayouts — открытая
  регистрация; редирект/партнёрка) вместо `mockleg`.
- Дизайн-система с темами/шрифтами/настройками; РФ-комплаенс (см. `COMPLIANCE.md`).
- Прод: rate-limit, `/security-review`, Helm/K8s, метрики.
