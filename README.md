# flight-meta

Метапоиск авиабилетов с **виртуальным интерлайном** (само-стыковка из отдельных
билетов) и **визово-транзитной логикой** под паспорта РФ/СНГ.

## Зачем

Агрегаторы (Aviasales и др.) показывают только маршруты, продающиеся как **единый
билет** в GDS. Они не «склеивают» два отдельных билета разных авиакомпаний —
поэтому реально существующие и часто более дешёвые маршруты (через любой хаб:
Стамбул, Дубай, Китай и т.п.) не видны. Этот сервис их находит.

Модель — **метапоиск без наценки пользователю**: находим, показываем, отправляем
покупать к авиакомпании/агентству (редирект). Доход — партнёрские отчисления.

Полный план и дорожная карта: `~/.claude/plans/greedy-snuggling-hummingbird.md`.

## Статус: Фаза 1 (фундамент)

Готов контейнерный Go-сервис: обобщённая модель оффера, интерфейс источника,
конкурентный fan-out с per-source таймаутом, фильтры (пересадки, авиакомпании,
само-стыковка), ранжирование по цене и **mock-источник** — чтобы конвейер
запускался и проверялся ещё до реальных ключей Kiwi/Travelpayouts.

## Запуск (локально)

```bash
go run ./cmd/api
# в другом окне:
curl "http://localhost:8080/health"
curl "http://localhost:8080/search?origin=MOW&destination=BKK&depart=2026-04-20"
```

Через Docker:

```bash
docker compose up --build
curl "http://localhost:8080/search?origin=MOW&destination=BKK&depart=2026-04-20"
```

## Фронтенд (web/)

React + Vite + TypeScript. Дизайн карточек — устоявшиеся паттерны flight-search
(Google Flights / Skyscanner): крупная цена, таймлайн маршрута, пилюли пересадок,
бейджи авиакомпаний. Само-стыковочные варианты помечаются («Само-стыковка»,
«Нет у агрегаторов») и сопровождаются предупреждением о раздельных билетах.

```bash
# 1) бэкенд
go run ./cmd/api
# 2) фронт (Vite dev проксирует /api → :8080)
cd web && npm install && npm run dev   # http://localhost:5173
```

Прод-сборка: `cd web && npm run build`. База API настраивается через `VITE_API_BASE`
(по умолчанию `/api`, в dev — прокси на бэкенд).

## API

`GET /health` → `{"status":"ok"}`

`GET /search` — параметры:

| Параметр | Обяз. | Пример | Описание |
|---|---|---|---|
| `origin` | да | `MOW` | IATA-код вылета (3 буквы) |
| `destination` | да | `BKK` | IATA-код назначения |
| `depart` | да | `2026-04-20` | дата вылета `YYYY-MM-DD` |
| `return` | нет | `2026-04-27` | дата обратно (one-way, если пусто) |
| `passengers` | нет | `2` | 1..9, по умолчанию 1 |
| `passport` | нет | `RU` | страна паспорта (для транзит-виз, Фаза 3) |
| `currency` | нет | `RUB` | валюта цен |
| `stops` | нет | `direct` / `one` / `one_plus` | фильтр пересадок |
| `airlines` | нет | `TK,EK` | только эти авиакомпании |
| `exclude_airlines` | нет | `SU` | исключить авиакомпании |
| `self_transfer` | нет | `false` | выключить само-стыковочные варианты |

Ответ: `{ offers: [...], sources: [{name,count,tookMs,error?}] }`. У само-
стыковочных офферов `connection: "self_transfer"` и часто `unique: true`.

## Структура

```
cmd/api            точка входа сервиса
internal/offer     единая модель Offer (без зависимостей)
internal/sources   интерфейс Adapter + Query/Filters; mock/ — заглушка-источник
internal/search    конкурентный fan-out, мердж, ранжирование
internal/rank      фильтры + сортировка
internal/config    настройки из env (секреты только server-side)
internal/httpapi   HTTP/JSON, валидация запроса, security-заголовки
deploy/            Dockerfile (далее — Helm для K8s)
```
